Sejarah Web Server

WebServer adalah sebuah aplikasi server yang melayani permintaan HTTP atau HtTPS dari browser dan mengirimkanya kembali dalam bentuk halaman-halaman web. Web server dibuat oleh Tim Barnes-Lee pada tahun 1989.Pada saat dia bekerja di perusahaan CERN (Europe Organization for Nuclear Research).Dia membuat sebuah projek berbasis tulisan hiperteks, yang bertujuan mempermudah pertukaran informasi diantara para peneliti. Satu tahun kemudian Tim Berners-Lee menghasilkan dua buah program komputer,yaitu

  • Sebuah sistemyang diberi nama WorldWideWeb atau disebut WWW.
  • Sebuah Web Server pertama di dunia yang diberi nama CERN httpd.

Perbedaan HTTP dan HTTPS

HTTP

HTTP adalah Hypertext Transfer Protokol.Sebagai bagian dari protokol. HTTP, data bergerak melaui jaringan, seperti internet.

  • HTTP mengirim pesan dalam teks biasa: Itu berarti siapa pun yang memiliki pengetahuan tentang perintah dan sintaks protokol HTTP dapat membaca dan memahaminya. Ini sangat berbahaya katika pengguna mungkin mengirimkan formulir, karena pengguna mungkin memberikan data sensitif, seperti kata sandi, nomor kartu kredit, atau nomorjaminan sosial.
  • Halaman web di situs yang menggunakan HTTP membuat lebih cepat daripada halaman web domain HTTPS: Ini karena HTTPS membutuhkan lebih banyak daya komputasi untuk mengenkripsi pesan.
  • Situs web yang mengunakan HTTP memiliki HTTP:// di awal Unique Resource Locators atau URL mereka.
  • Browser internet menghasilkan permintaan saat pengguna internet berinteraksi dengan properti web. Contoh, browser akan menghasilkan serangkaian permintaan "HTTP GET" ketika pengguna internet menekan tombol hyperlink.
HTTPS

S dalam HTTPS berarti "secure", HTTPS adalah singkatan dari Hypertxt Transfer Protocol secure. Situs Web yang menggunakan HTTPS memiliki https:// di awal URL mereka, dan browser dapt menandai URl ini dengan meletakan gembok hijau di depan.

  • HTTPS mempromosikan keamanan karena menampilkan sekumpulan karakter yang tampak acak, bukan tekt biasa: Karakter dalam permintaan dan respons HTTPS berisi campuran huruf kecil, huruf besar, dan simbol tanpa spasi atau pengambalian.
  • HTTPS bergantung pada otentikasi: Browser harus memverifikasi bahwa seseorang, mesin, atau situs web adalah siapa atau apa yang mereka klaim. HTTP hanya didasarkan pada prinsip kepercayaan.
  • Setiap situs web yang memerlukan kredensial pengguna dan informasi sensitif perlu menggunakan enkripsi
  • HTTPS menggunakan HTTP, tetapi juga menggunakan protokol Transport Layer Security (TLS) untuk mengenkripsi permintaan dan tanggapan HTTPS

Jenis-jenis web server dan penjelasanya

Pengertian Web Server

Web server adalah sejenis perangkat lunak yang mempu memberikan layanan atau service untuk menerima request HTTP maupun HTTPS dari klien yang dikenal seperti Google Chrome, Mozila Firefox, UC Browser, dan lain sebagainya. Setelah menerima kembali request dari klien, otomatis web server akan mengirimkan kembali request yang diminta oleh klien dalam bentuk halaman web, yang umumnya akan berbentuk dokumen HTML.

Jenis-jenis Web Server
  • Apache

    Web Server untuk sistem operasi UNIX, dengan beberapa dukungan kontrol akses, PHP, SSL.

  • Nginx

    Nginx dapat menangani kebutuhan web server dengan trafik yang sangat tinggi, sekaligus mengedepankan performa, kemudahan, dan kecepatan.

  • IIS (Internet Information Services)

    Windows versi lawas, seperti Windows 2000 dan Windows 2003 umumnya akan menggunakan web server berjenis IIS. Web IIS didukung banyak macam fitur dan konponen seperti TCP/IP, DNS, hingga software untuk membuat situs web. Dan IIS merupakan satu-satunya web yang mendukung platform .NET dari windows.

  • Lighttpd

    Lighttpd atau Lightly merupakan web server yang hanya dapat bekerja pada komputer dengan sistemm operasi Linux dan Unix beserta turunanya.

Cara setting IIS di Windows



1. Klik search dan ketik "Turn Windows features on or off" pada control panel.


2. Kemudian pada tampilan "Windows Features" scrol ke bawah untuk pilihan Cek pada Internet Information Services yang menandakan kita akan mengaktifkan IIS pada Windows 10. Selanjutnya klik OK dan tunggu proses sampai selesai.


3. Setelah proses selesai cek di search windows jika sudah aktif akan tampil seperti gambar dan aplikasi akan muncuk di start menu windows
